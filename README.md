# 🚀 Corporate RAG Assistant API (Proof of Concept)

Микросервис на базе **FastAPI** и **RAG (Retrieval-Augmented Generation)** для интеллектуального поиска ответов по корпоративным регламентам и базам знаний. 

Проект создан для демонстрации того, как ИИ может автоматизировать работу операторов CRM, предоставляя мгновенные и точные ответы строго на основе загруженных документов (без "галлюцинаций").

## 💼 Бизнес-ценность
* **Экономия времени:** Операторам не нужно искать информацию по PDF-файлам и инструкциям.
* **Точность:** Модель отвечает только по предоставленному контексту. Если ответа нет в базе — она честно об этом говорит.
* **Достоверность:** Каждый ответ сопровождается ссылкой на конкретный пункт регламента (`sources`).

## 🛠 Технологический стек
* **Backend:** Python, FastAPI, Uvicorn, Pydantic
* **LLM Integration:** Groq API (Llama-3.1-8b-instant), AsyncOpenAI
* **Vector Database:** Qdrant (in-memory)
* **Embeddings:** SentenceTransformers (`paraphrase-multilingual-MiniLM-L12-v2`)
* **Environment:** Google Colab, Localtunnel (для внешнего доступа)

## ⚡️ Ключевые фичи
1. **Векторизация "на лету":** Тексты превращаются в эмбеддинги и загружаются в векторную БД Qdrant.
2. **Семантический поиск:** Использование Cosine Similarity для нахождения наиболее релевантных кусков текста по запросу пользователя.
3. **Безопасность:** Полное отсутствие захардкоженных API-ключей. Использование менеджера секретов среды.

## 🚀 Запуск проекта (Google Colab)
Проект реализован в формате интерактивного Jupyter Notebook для быстрого деплоя и тестирования.

1. Откройте файл `rag_corporate.ipynb` в Google Colab.
2. В боковом меню Colab (вкладка "Секреты" / 🔑) добавьте свой ключ от Groq API с названием `apigrok` и включите доступ к нему для ноутбука.
3. Запустите все ячейки.
4. В последней ячейке будет сгенерирована публичная ссылка (через Localtunnel), по которой можно отправлять POST-запросы на `/api/ask`.
